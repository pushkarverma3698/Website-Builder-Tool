How should we architect the 3D site?

Three valid approaches — each trades immersion against maintainability.

OPTION B

Hybrid — Fixed Canvas + Overlay HTML

✓ Recommended

A fullscreen R3F canvas is pinned as a fixed background layer. Standard React DOM sections scroll over it. A single GSAP ScrollTrigger timeline reads scroll position and drives camera movement + particle state in the 3D world. HTML sections fade in/out as the camera moves to new "scenes".

✓ PROS

  • Accessible HTML text for SEO/screen readers
  • Easy to style 2D sections independently
  • Standard DOM layout — no 3D positioning headaches
  • Industry standard (most top-tier 3D sites use this)
  • Degrades gracefully if WebGL fails

✗ CONS

  • Z-index layer management needed
  • Less "total immersion" than pure canvas
HTML sections (scroll) z-index layering ↓ R3F Canvas (fixed bg) ↕ synced via GSAP ScrollTrigger
OPTION A

Pure R3F Canvas — Page IS the 3D World

The entire page is one Three.js canvas. HTML content is rendered inside the 3D scene using drei's <Html> component. All 6 scenes are "rooms" in 3D space. Camera flies between them. Most cinematic possible experience.

✓ PROS

  • Maximum cinematic immersion
  • True 3D depth — text exists in 3D space
  • Unified render loop — no sync complexity

✗ CONS

  • Heavy mobile GPU load (1000 nodes risk)
  • SEO / accessibility significantly harder
  • Three.js Html component has layout quirks
  • Harder to iterate and debug
R3F Canvas (full viewport) ↓ inside the canvas <Html> text overlays + 3D geometry + camera path
OPTION C

Per-Section Embedded 3D Windows

Standard scrollable HTML page. Each section has its own isolated <Canvas> component as a visual element. No camera path — each 3D piece stands alone. Simplest to build and extend.

✓ PROS

  • Simplest architecture — easiest to extend
  • Each 3D piece is independent + testable
  • Best mobile performance

✗ CONS

  • Not cinematic — feels like "cards with decoration"
  • No immersive scroll journey
  • Multiple canvases = more GPU overhead than 1
HTML section 1 Canvas A (hero) HTML section 2 Canvas B (features) HTML section 3...

💡 Option B is what Lusion.co, Bruno Simon's portfolio, and most top 3D studios use in production. It's the right call for a demo that needs to run on any device.
