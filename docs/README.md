# docs/

Reference documentation for the cinematic-web project.

```
docs/
├── phases/          # Phase tracking — one doc per phase, always kept up to date
│   ├── PHASE-1-CLI.md          ✅ Complete
│   ├── PHASE-2-PORTFOLIO.md    🔄 In progress
│   └── PHASE-3-SAAS.md         ⬜ Not started
├── architecture/    # Technical architecture decisions, data flows, module boundaries
│   └── (see PHASE docs for current architecture; dedicated docs added as system grows)
├── study/           # Research, competitive analysis, market data
│   └── competitive-analysis.md
└── superpowers/     # Auto-generated by superpowers skills (plans, specs)
```

## Convention

- **Phase docs** (`phases/PHASE-X-NAME.md`) are living documents — update status and decisions as each phase progresses.
- **Architecture docs** (`architecture/`) capture stable decisions: data flows, module boundaries, DB schema, API contracts.
- **Study docs** (`study/`) are reference material: competitive research, market sizing, user research.
- **Never leave docs flat** — always file into the appropriate folder.
